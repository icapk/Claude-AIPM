# 第 0 章：AI 产品经理的四个段位

> 在开始之前，先看清你在哪、要去哪。

## 段位矩阵

| | 🥉 L1 套壳 PM | 🥈 L2 特性 PM | 🥇 L3 架构 PM | 💎 L4 引擎 PM |
|---|---|---|---|---|
| **典型产出** | 包一层 ChatGPT UI | 设计 RAG/搜索增强功能 | 设计完整 Agent 产品架构 | 定义 Agent 引擎的产品规则 |
| **对 AI 的理解** | "调 API 得结果" | "检索+生成，能调参" | "循环执行，工具驱动" | "理解引擎，能改规则验证想法" |
| **与工程师沟通** | "帮我接个 API" | "这个 RAG 召回率不够" | "循环边界设 20 轮，压缩阈值 85%" | "改这行 prompt 看行为变化" |
| **成本认知** | "按 API 调用次数算" | "知道 token 概念" | "算得清上下文重传 + 压缩开销" | "能预估一次完整任务的 token 消耗" |
| **安全认知** | "加个内容审核" | "敏感词过滤 + 人工审核" | "权限分级 + 操作确认" | "设计分层信任体系" |
| **核心局限** | 做不了差异化 | 做不了自主 Agent | 需要工程师翻译想法 | — |

## 段位详解

### 🥉 L1：套壳 PM

**标志：** 接一个 AI API，套个前端 UI，就觉得自己在做 AI 产品。

**常见场景：**
- "我们做个聊天机器人吧"
- "把 GPT 接到我们的客服系统里"
- "用户输入 → 调 API → 显示结果"

**瓶颈：** 产品没有差异化——你能做的，别人换个 API Key 也能做。

---

### 🥈 L2：特性 PM

**标志：** 知道 RAG、embedding、fine-tuning 这些概念，能做参数调优相关的产品决策。

**常见场景：**
- "我们的知识库检索召回率太低，要不要换向量数据库？"
- "这个场景用 claude-sonnet 就够了，不需要 opus"
- "给用户加个 temperature 调节"

**瓶颈：** 做不了自主 Agent——只会「一问一答」，不会「给一个任务让 AI 自己干」。

---

### 🥇 L3：架构 PM

**标志：** 理解 Agent 的循环本质，能设计工具集、安全分级、成本模型。

**常见场景：**
- "这个 Agent 最多循环 20 轮，超过就停"
- "文件写操作要确认，读操作自动执行"
- "Token 快满了触发自动压缩，阈值设 85%"
- "给 Agent 6 个工具，每个工具的描述要让模型准确选择"

**瓶颈：** 有想法但需要工程师翻译成代码才能验证。

---

### 💎 L4：引擎 PM

**标志：** 能直接改配置和提示词来验证产品假设，不需要等工程师排期。

**常见场景：**
- "我改了一行 System Prompt，Agent 行为变了——验证了我的假设"
- "加了一句工具描述，模型选错工具的概率从 30% 降到 5%"
- "通过 CLAUDE.md 让用户自己教会 Agent 新规则"
- "我能预估这个任务场景的 token 消耗，定价不会亏"

**这是本教程的目标。**

---

## 每个段位的学习重点

| 目标段位 | 必学章节 | 预计时间 | 核心能力 |
|---|---|---|---|
| **L1 → L2** | 第 1-2 章 | 2 小时 | 理解 Agent 循环 + 看到工具调用 |
| **L2 → L3** | 第 3-5 章 | 4 小时 | 设计工具集 + 提示词架构 + 成本模型 |
| **L3 → L4** | 第 6-8 章 | 4 小时 | 安全设计 + 生产认知 + 动手验证 |

> 💡 **建议：** 即使你觉得自己已经是 L2 或 L3，也请从第 1 章开始——**跑通 Agent 的体验无法替代**。
