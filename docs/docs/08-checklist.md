# 第 8 章：实战武器库

> **段位目标：L4** | ⏱️ 60 分钟 | 终极实战

## 8.1 Agent 产品设计 Checklist

**每次设计 Agent 产品或功能时，过一遍这个清单：**

### 🔄 自主性设计

- [ ] 最大循环步数是多少？为什么是这个数？
- [ ] 预算上限是多少？按 token 还是按金额？
- [ ] 超时设置？Agent 卡死了能退出吗？
- [ ] 用户中断后，中间结果怎么处理？丢弃 / 保留 / 可选？
- [ ] 任务失败后重试几次？间隔多久？

### 🔧 工具/能力设计

- [ ] Agent 有哪些工具？列出完整清单
- [ ] 每个工具的描述够好吗？模型能正确选择吗？
- [ ] 工具超过 15 个了吗？需要发现/分组机制吗？
- [ ] 工具结果太大怎么办？截断阈值是多少？
- [ ] 需要支持外部工具（MCP 协议）吗？

### 📝 提示词架构

- [ ] 分了几层？基座 / 环境注入 / 用户自定义 / 动态上下文
- [ ] 用户能自定义规则吗？（CLAUDE.md 式机制）
- [ ] 提示词变化会破缓存吗？缓存策略是什么？
- [ ] 不同场景有不同的提示词吗？

### 💰 成本模型

- [ ] 一次典型任务消耗多少 token？（实际值，不是猜测）
- [ ] 成本放大倍数是多少？（通常 3-10x）
- [ ] 压缩策略有几层？触发阈值是多少？
- [ ] 定价覆盖了压缩成本和工具调用吗？
- [ ] 给用户看实时消耗吗？

### 🛡️ 信任设计

- [ ] 安全至少分 3 档？（保守 / 标准 / 信任）
- [ ] 默认是最安全的档位？
- [ ] 升级信任需要明确确认？
- [ ] 有不可降级的底线操作吗？
- [ ] 权限有记忆吗？（不重复问同一个路径）
- [ ] 错误操作能撤销吗？

### 🔗 集成与扩展

- [ ] 独立产品还是嵌入已有工具？
- [ ] 需要多 Agent 协作吗？怎么拆分任务？
- [ ] 需要插件/技能生态吗？
- [ ] 需要 feature flags 做灰度吗？

## 8.2 实战练习：设计你自己的 Agent

用上面的 Checklist，完整设计一个 Agent 产品。

### 示例：自动写周报的 Agent

**工具集：**

| 工具 | 说明 | 风险 |
|---|---|---|
| `read_calendar` | 读取本周日历事件 | 无 |
| `read_commits` | 读取本周 Git 提交 | 无 |
| `read_tasks` | 读取任务管理系统已完成项 | 无 |
| `write_document` | 生成周报文档 | 低 |
| `send_email` | 发送邮件 | **高** |

**循环边界：**
- 最大 10 步（读 3 个数据源 + 生成文档 + 发送 = 5 步，留一倍余量）
- 超过 50K token 触发轻量压缩
- 超时 3 分钟自动停止

**安全分级：**
- 读操作 → 自动执行
- 写文档 → 自动执行（可覆盖旧周报）
- 发邮件 → **必须确认**（不可自动化的底线操作）

**提示词架构：**
- 基座："你是一个周报助手，帮用户整理本周工作并生成格式化周报"
- 环境注入：当前日期、用户姓名、部门
- 用户自定义：`WEEKLY_REPORT_TEMPLATE.md`（周报模板）

**成本估算：**
- 5 步 × 平均每步 3K input token = ~15K input
- 累积发送：3+6+9+12+15 = ~45K input total
- Output: ~5K
- 单次成本（Claude Sonnet）：~$0.02

## 8.3 五个心智模型

学完全部 8 章后，你应该内化这 5 条认知：

### 🧠 模型 1：Agent 是循环，不是单次调用
用户说一句话，Agent 可能执行 3-20 步。每一步都是产品行为，都要设计边界。

### 🧠 模型 2：能力由工具集定义，描述质量 > 工具数量
模型靠 description 文本选工具。6 个描述精确的工具胜过 20 个描述含糊的。

### 🧠 模型 3：上下文有限，压缩有成本
每轮重发全部历史，成本累积增长。压缩花额外钱，但可以省更多。

### 🧠 模型 4：安全需要分级，不是开关
至少 3 档。默认保守，高级用户可手动升级，底线操作不可跳过。

### 🧠 模型 5：真实成本 = 你以为的 × 3~5 倍
定价前一定要用真实 token 消耗计算，不要拍脑袋。
